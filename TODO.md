## 阶段三：软标签（Soft Labels）
**目标**：用事件覆盖比例（0.0~1.0）替代二值标签，减少标注边界噪声。
**工作内容**：
- [ ] 运行 `prep_mesa_enhance.py --soft-label`，生成软标签数据集
- [ ] 确认 `runner.py` 中损失函数为 `BCEWithLogitsLoss`（支持 float 标签）
- [ ] 训练，评估时预测值 > 0.5 视为正类
**对应 exp_tag**：
- `delay{D*}_win{W*}_soft1`
**代码改动**：约 1 个文件（`runner.py` loss 兼容性确认）。
**难点**：确认模型输出和 loss 兼容 float 标签（而非 int）。
**关于二值化**：软标签（0.0~1.0）仅用于训练时的 loss 计算，让模型学到事件覆盖程度的连续信息。评估指标（准确率、F1、混淆矩阵）只能处理离散的 0/1，因此评估时需将软标签以 0.5 为阈值转回二值（≥0.5 → 1，< 0.5 → 0）。简言之：**软标签用来训练，二值化用来评估**。

### ML备注
1. **ML 的 F1 和 Sens 反超 DL**（F1: 68.83 vs 65.49 最高；Sens: 72.71 vs 68.36 最高）——但这并不能说明 "ML 优于 DL"，原因是两者的任务设定本质不同：
   - ML 做的是"事件中心对齐的闭集二分类"，正样本都是事件中心，负样本是明确无事件段，任务边界清晰
   - DL 做的是"连续滑窗检测"，需要处理事件边界模糊、部分覆盖等真实场景
   - 论文撰写时应明确区分两者的任务定位，避免误读
2. **DecisionTree 表现显著落后**（AUROC 57.72 接近随机），符合单棵树处理高维连续特征的理论缺陷，反向说明了 RandomForest 集成的必要性。
3. **SVM_RBF 与 RandomForest 性能相当**（Acc 均 ~67%、AUROC 均 ~73%），说明"7 通道形态特征 × 7 统计量聚合"这套 49 维特征的表达能力在该水平触顶。
4. **ML 的 Pearson r 偏低（0.12~0.20）**：事件中心 + 1:1 平衡采样使每个受试者的正样本率恒为 50%，无法反映真实 AHI 分布，该指标在 ML 设定下参考价值有限；DL 的滑窗阳性率才和真实 AHI 线性相关。
